## Задание №6.

[Ссылка на отчёт](https://colab.research.google.com/drive/178Kz90W7Y_qZFON2yRHhCoRP_bulbZ0L?authuser=0#scrollTo=8AVGKHi60R8P)

Итог работы:
В результате проектирования была создана база данных SportsShop, состоящая из четырёх таблиц, каждая из которых находится в 3НФ:

Customer (покупатели) — хранит данные о клиентах: уникальный номер, имя, адрес, телефон.
Product (товары) — хранит информацию о товарах: каталожный номер, название, цену.
Orders (заказы) — хранит заголовки заказов: номер заказа, номер покупателя, дату.
OrderDetail (состав заказа) — хранит позиции заказов: номер заказа, каталожный номер товара, количество.
Все неключевые атрибуты зависят только от первичного ключа своей таблицы и ни от каких других полей — это главный признак 3НФ. Например, адрес и телефон клиента хранятся только в таблице Customer и не дублируются в каждом заказе. Если клиент меняет телефон, его достаточно обновить в одном месте.

Связи между таблицами:

Один покупатель может сделать много заказов (Customer → Orders — 1:N)
Один заказ может содержать много позиций (Orders → OrderDetail — 1:N)
Один товар может встречаться во многих заказах (Product → OrderDetail — 1:N)
Учтены различия между MySQL и Microsoft Access:

Вместо TOP (как в Access) в MySQL используется LIMIT для ограничения количества строк.
Синтаксис LIKE для поиска по шаблону работает в обеих СУБД, но в отчёте отмечены нюансы его использования.
